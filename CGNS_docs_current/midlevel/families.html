Input/Output

    fn   CGNS file index number.
B Base index number, where 1 ≤ B ≤ nbases.
nfamilies Number of families in base B.
Fam Family index number, where 1 ≤ Fam ≤ nfamilies.
FamilyName Name of the family.
NodeName Name of the FamilyName_t node. FamilyParent is used to refer to the parent family of the Family_t node.
nFamBC Number of boundary conditions for this family. This should be either 0 or 1.
nGeo Number of geometry references for this family.
nNames Number of FamilyName_t nodes for this family.
N Family name index number, where 1 ≤ N ≤ nNames.
ier Error status.

Input/Output

    fn   CGNS file index number.
B Base index number, where 1 ≤ B ≤ nbases.
Fam Family index number, where 1 ≤ Fam ≤ nfamilies.
G Geometry reference index number, where 1 ≤ G ≤ nGeo.
P Geometry entity index number, where 1 ≤ P ≤ nparts.
GeoName Name of GeometryReference_t node.
FileName Name of geometry file.
CADSystem Geometry format.
nparts Number of geometry entities.
PartName Name of a geometry entity in the file FileName.
ier Error status.

Note that with cg_geo_read the memory for the filename character string, FileName, will be allocated by the Mid-Level Library. The application code is responsible for releasing this memory when it is no longer needed by calling cg_free(FileName).

Note that the family name written cg_famname_write is stored in a FamilyName_t node named FamilyName as in previous versions. The additional family names written with cg_multifam_write are stored in AdditionalFamilyName_t nodes.

    fn   CGNS file index number.
B Base index number, where 1 ≤ B ≤ nbases.
Fam Family index number, where 1 ≤ Fam ≤ nfamilies.
sn Solution Interpolation index number, where 1 ≤ sn ≤ nSolutionInterpolations.
    sinterpName   Node name.
et Element Type.
os Spatial order of interpolation.
ot Temporal order of interpolation.
nSolutionInterpolations Number of solution Interpolation nodes.
it Name of the Interpolation basis.
pu pointer to 1st parametric coordinates. Needs to be allocated if wanted. See cg_solution_lagrange_interpolation_size.
pv pointer to 2nd parametric coordinates. Needs to be allocated if wanted. See cg_solution_lagrange_interpolation_size.
pw pointer to 3rd parametric coordinates. Needs to be allocated if wanted. See cg_solution_lagrange_interpolation_size.
pt pointer to the temporal parametric location. Needs to be allocated if wanted. See cg_solution_lagrange_interpolation_size.
sz Number of parametric nodes required for the specified triplet (Element type, spatial order, temporal order).
cnt Number of solution Interpolation nodes corresponding to the specified Element type (i.e. its cardinality). Should be 0 or 1.
ier Error status.

Note that cg_solution_interpolation_points_read returns CG_NODE_NOT_FOUND if the InterpolationPoints node does not exists.
